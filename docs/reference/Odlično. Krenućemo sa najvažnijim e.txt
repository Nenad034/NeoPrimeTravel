Odlično. Krenućemo sa **najvažnijim ekranom celog sistema** — **centralna forma rezervacije (Reservation Dossier / Booking File)**.
To je ekran koji operateri koriste **ceo dan**, zato mora biti:

* brz
* pregledan
* modularan
* bez previše klikova
* sa jasnom hijerarhijom informacija

U modernim travel sistemima to se zove **Reservation Dossier**, **Booking File** ili **Reservation Architect**.

Ja predlažem **modularni layout** koji može da prikaže sve informacije bez haosa.

---

# 1. Koncept forme rezervacije

Ekran mora imati **tri nivoa informacija**:

### 1️⃣ Header (status rezervacije)

Najvažnije informacije odmah vidljive.

### 2️⃣ Main workspace

Radni deo gde operater menja rezervaciju.

### 3️⃣ Sticky finansijski panel

Pregled novca i statusa plaćanja.

---

# 2. Predloženi layout

```text
-------------------------------------------------------------
BOOKING HEADER
-------------------------------------------------------------

-------------------------------------------------------------
LEFT SIDEBAR     | MAIN WORKSPACE                | FINANCE
Navigation       | Reservation data              | PANEL
                 |                               |
                 |                               |
                 |                               |
-------------------------------------------------------------
```

---

# 3. Header rezervacije

Header mora odmah pokazati **status rezervacije**.

Primer:

```
Reservation: OT-2026-000482
Status: CONFIRMED
Channel: B2B Agent
Agent: TravelPro Belgrade
Created: 14.03.2026
Operator: Marko Jovanović
Travel date: 10.07.2026
```

### Status badge

Boje statusa:

```
DRAFT        gray
OPTION       amber
CONFIRMED    green
CANCELLED    red
ON REQUEST   blue
PROBLEM      orange
```

### Brze akcije u headeru

Desno u headeru:

```
Edit reservation
Cancel reservation
Send voucher
Send invoice
Add note
```

# 16. Prednosti ovakvog dizajna

Operater može:

* pratiti novac
* generisati dokumente

# 18. Sledeći korak

Ako želite, sledeće mogu da uradim:

1️⃣ **kompletan dizajn finansijskog modula**
2️⃣ **operativne liste (arrival / departure / transfer)**
3️⃣ **kapacitet i allotment modul**
4️⃣ **rooming drag-and-drop interfejs**

To su sledeće ključne komponente operativnog sistema.
