# IMPLEMENTACIONI BLUEPRINT

## Kako ovu platformu pretvoriti u realan, izvodljiv plan razvoja

Sada više ne pričamo samo o arhitekturi, već o:

* redosledu izrade
* zavisnostima između modula

* realnim fazama
* MVP granicama

* prioritetima

# 1. Osnovni princip implementacije

Ovako veliki sistem ne treba graditi “sve odjednom”.

Treba ga graditi po principu:

## 1.1 Core first

Prvo ono bez čega nema realnog rada:

* korisnici
* CRM
* proizvodi
* ugovori
* search
* booking
* finansije
* osnovna operativa

## 1.2 Operations before fancy AI

Pre AI i naprednog marketinga mora raditi:

* rezervacija
* rooming
* dokumenta
* uplata/isplata
* operativne liste
* B2B rad

## 1.3 AI kao kontrolisani sloj

AI ulazi tek kada su:

* podaci čisti
* workflow stabilan
* permission model jasan
* trošak kontrolisan

# 15. TEHNIČKI RIZICI PROJEKTA

Ovo je vrlo važno.

## 15.1 Najveći rizici

### 1. Supplier integration complexity

Različiti API-i, loš kvalitet odgovora, različiti modeli cena.

### 2. Pricing complexity

Occupancy, supplements, promotions, package logic, charter/fixed/hybrid model.

### 3. Booking orchestration

Posebno kada imate više item-a i mixed sources.

### 4. Finance correctness

Receivables, payables, margin, refunds, agency credit.

### 5. Rooming / charter / allotment logic

Vrlo lako nastane haos ako model nije strog.

### 6. Scope explosion

Sistem je ogroman i lako “pojede sam sebe” ako nema discipline.
