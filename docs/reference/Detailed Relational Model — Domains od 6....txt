Ovo je centralni transakcioni domen cele platforme.

* OTA booking
* tour operator booking
* charter booking

## 6.27 Poslovna pravila

1. Booking reference mora biti jedinstven globalno.
2. Booking može imati više items, ali item_no mora biti jedinstven unutar booking-a.
3. Passenger snapshot se ne sme menjati bez audit zapisa posle potvrde.
4. Core item failure može oboriti ceo booking, optional item failure ne mora.
5. Booking document može biti na nivou booking-a ili pojedinačnog item-a.
