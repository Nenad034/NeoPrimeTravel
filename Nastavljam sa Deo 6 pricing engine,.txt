# TEHNIČKI DOKUMENT ZA RAZVOJNI TIM

## Unified Travel Booking Platform

### Deo 6 — Pricing engine, commercial rules i package ranking logika

# 1. Svrha pricing engine-a

Pricing engine mora da radi 6 stvari:

1. preuzme net cenu iz dobavljačkog izvora
2. doda takse, fees i doplate
3. primeni markup i commission logiku
4. primeni promocije i bundle discount
5. vrati finalnu sell cenu po kanalu
6. sačuva pricing snapshot za booking i audit

Pricing engine mora biti:

* centralizovan
* dosledan
* auditabilan
* fleksibilan za B2C i B2B

---

# 2. Ključni komercijalni pojmovi

## 2.1 Net cena

Cena koju vraća supplier ili ugovorena cena.

## 2.2 Gross / sell cena

Cena koju vidi kupac ili agent.

## 2.3 Markup

Razlika koju sistem dodaje na net cenu.

## 2.4 Commission

Iznos koji ide agentu ili prodajnom kanalu.

## 2.5 Service fee

Dodatna operativna taksa.

## 2.6 Bundle discount

Popust koji se daje kada se više proizvoda prodaje zajedno.

## 2.7 Promo discount

Akcijski ili kuponski popust.

---

# 3. Opšta pricing formula

## 3.1 Osnovna formula po item-u

```text
sell_price =
net_price
+ taxes
+ fees
+ markup
- item_discount
```

## 3.2 Formula za paket

```text
package_total =
sum(all_item_sell_prices)
- bundle_discount
- promo_discount
```

## 3.3 Formula za B2B model sa provizijom

```text
agent_sell_price =
net_price
+ platform_markup
```

```text
agent_commission =
agent_sell_price * commission_rate
```

Ili, alternativno, ako agent radi sa net cenom:

```text
agent_net_price = net_price + agreed_markup
```

# 6. Pricing scope i prioritet pravila

Pravila se mogu definisati na više nivoa.

## 6.1 Nivoi pravila

* global
* sales channel
* product type
* supplier
* destination
* hotel / airline / cruise line
* agent / agent group
* booking date
* travel date
* campaign

## 6.2 Predloženi redosled prioriteta

Od najspecifičnijeg ka najopštijem:

```text
agent-specific
→ product-specific + destination-specific
→ supplier-specific
→ channel-specific
→ global default
```

## 6.3 Primer

Ako postoji:

* global hotel markup 10%
* B2C hotel markup 12%
* Paris hotel markup 15%
* Agent X hotel markup 8%

onda sistem za Agent X u Parizu mora primeniti ono pravilo koje je definisano kao najprioritetnije za taj scope.

# 7. Tipovi commercial pravila

## 7.1 Markup pravila

Podržati:

* fixed amount
* percentage
* stepped markup
* minimum markup
* maximum markup

### Primer percentage

```text
hotel markup = 12%
```

### Primer fixed

```text
flight service fee = 15 EUR
```

### Primer stepped

```text
if net_price < 100 → markup 15 EUR
if net_price 100–500 → markup 10%
if net_price > 500 → markup 7%
```

---

## 7.2 Commission pravila

Podržati:

* fixed commission
* percentage commission
* tiered commission

### Primer

```text
agent commission = 7%
```

ili:

```text
if monthly_sales > 50,000 EUR → commission 9%
else commission 6%
```

# 8. B2C pricing logika

B2C kanal treba da koristi:

* prikazanu finalnu cenu
* market-oriented markup
* promocije
* package savings komunikaciju

## 8.1 B2C formula

```text
b2c_sell =
supplier_net
+ taxes
+ b2c_markup
+ fees
- b2c_discounts
```

## 8.2 B2C UI prikaz

Kupcu treba prikazati:

* total price
* currency
* savings if package
* optionally “from” price
* cancellation condition
* included/not included

Ne treba nužno prikazivati:

* net price
* internu maržu
* supplier info ako to nije deo poslovnog modela

---

# 9. B2B pricing logika

B2B kanal može raditi na više modela.

## 9.1 Model A — bruto cena + agent provizija

Agent vidi prodajnu cenu i svoju proviziju.

## 9.2 Model B — net cena za agenta

Agent dobija net cenu i sam dodaje svoj markup.

## 9.3 Model C — ugovoreni markup model

Platforma vraća cenu već prilagođenu agentu.

Za vas je najpraktičnije da sistem podrži bar:

* B2C gross model
* B2B contracted sell/net model

---

## 9.4 B2B formula primer

```text
b2b_price =
supplier_net
+ b2b_markup
- agent_discount
```

ili:

```text
b2b_sell =
supplier_net
+ markup
```

```text
agent_commission =
b2b_sell * commission_rate
```

# 10. Payment-dependent pricing pravila

# 17. Package discount engine

Paketni popusti ne treba da budu ad hoc, već sistemski.

## 17.1 Tipovi package discount-a

* hotel + flight only
* hotel + flight + transfer
* hotel + flight + activity
* cruise + flight
* destination-specific package discount
* supplier-funded package discount
* opaque hotel package discount

## 17.2 Pravila

Popust može biti:

* fixed amount
* percent
* conditional based on total basket
* conditional based on product combination

## 17.3 Primeri

### Primer 1

```text
if package contains hotel + flight:
   discount = 5%
```

### Primer 2

```text
if destination = PAR and package contains theme_park_activity:
   discount = 30 EUR
```

### Primer 3

```text
if hotel supplier = direct_contract and sold with flight:
   apply hidden hotel discount
```

# 18. Savings logika

Ako prikazujete “ušteda”, sistem mora jasno znati kako se računa.

## 18.1 Predlog formule

```text
savings =
(sum of standalone comparable sell prices)
- package sell price
```

## 18.2 Pravilo

Standalone comparison mora biti baziran na:

* istoj sobi
* istim datumima
* istoj tarifi
* istim passenger parametrima

Inače je savings prikaz netačan.

---

# 19. Supplier priority rules

Pricing i ranking zavise i od prioriteta supplier-a.

## 19.1 Zašto

Isti hotel ili sličan flight može doći iz više izvora.

Sistem mora znati:

* koga prvo koristi
* koga koristi za bolju cenu
* koga koristi za veću maržu
* koga koristi za veću pouzdanost

## 19.2 Faktori prioriteta

* niža net cena
* bolja marža
* bolja stopa potvrde
* bolja brzina API-ja
* dopušten package usage
* manji cancellation risk

## 19.3 Supplier score formula primer

```text
supplier_offer_score =
0.35 * margin_score
+ 0.25 * price_score
+ 0.20 * reliability_score
+ 0.10 * response_time_score
+ 0.10 * package_eligibility_score
```

# 21. Price change handling

Ako se cena promeni između search i booking faze, sistem mora imati pravilo.

## 21.1 Price up

Ako je nova cena viša:

* stop booking flow
* prikaži novu cenu
* traži potvrdu korisnika

## 21.2 Price down

Ako je nova cena niža:

* može automatski ažurirati snapshot
* ili prikazati korisniku novu bolju cenu

Za UX je najbolje:

* oba slučaja jasno prikazati

# 22. Package ranking engine — svrha

Posle pricing-a, sistem mora odlučiti koji paketi izlaze prvi.

Cilj nije samo najniža cena, već balans:

* verovatnoća kupovine
* kvalitet ponude
* profitabilnost
* operativna pouzdanost

---

# 23. Ranking signali

Predlažem sledeće glavne signale.

## 23.1 Price competitiveness

Koliko je cena dobra u poređenju sa sličnim opcijama.

## 23.2 Margin score

Koliko je paket profitabilan.

## 23.3 Product quality score

Ocena hotela, itinerary kvaliteta, cruise kvaliteta, activity popularnosti.

## 23.4 Convenience score

Direktan let, dobro vreme dolaska, jednostavan transfer, dobar check-in alignment.

## 23.5 Flexibility score

Refundable opcije, bolji cancel uslovi.

## 23.6 Supplier reliability

Koliko je supplier stabilan.

## 23.7 Conversion likelihood

Istorijski performance sličnih paketa.

## 23.8 Popularity score

CTR, bookings, add-to-cart ponašanje.

---

# 24. Predlog ranking formule

```text
package_score =
0.22 * conversion_score
+ 0.18 * margin_score
+ 0.16 * quality_score
+ 0.14 * price_competitiveness
+ 0.10 * convenience_score
+ 0.08 * flexibility_score
+ 0.07 * supplier_reliability
+ 0.05 * popularity_score
```

Težine će business tim kasnije fino podešavati.

---

# 25. Price competitiveness score

Treba da meri koliko je paket dobar u odnosu na tržište unutar istog rezultata.

Primer:

* najniža cena dobija viši score
* ali ne automatski 100, jer možda ima loš hotel ili loš flight

Formula može biti relativna prema median/lowest u rezultatu.

---

# 26. Margin-aware ranking

Ne preporučujem da prikaz bude isključivo po marži, ali marža mora biti signal.

## Pravilo

Paket sa neznatno višom cenom, ali mnogo boljom maržom i istim kvalitetom, može dobiti viši rang.

Ovo je veoma važno za poslovni rezultat platforme.

---

# 27. Diversity pravila u ranking-u

Bez diversity logike rezultat može biti loš:

* 10 skoro istih paketa iz istog hotela
* sve opcije iste aviokompanije
* nema raspona cena

## Predlažem ograničenja

* max 3 paketa po hotelu u top 20
* max 5 paketa po airline family
* obavezna zastupljenost price band-ova:

  * budget
  * mid
  * premium

---
